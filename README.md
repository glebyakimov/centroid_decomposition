# Бенчмарк: ближайшая красная вершина на дереве

Сравнение алгоритмов для задачи **динамических красных вершин** на дереве: после серии операций `paint` и `query` нужно быстро отвечать на запрос «расстояние от вершины `v` до ближайшей красной».

Проект связан с постановкой из презентации по **центроидной декомпозиции** (nearest active / красные вершины, Prüfer-like деревья). Здесь собран воспроизводимый C++-бенчмарк, скрипт усреднения по seed и сводные графики.

---

## Постановка задачи

Дано дерево на `n` вершинах (нумерация `0 … n-1`).

| Операция | Смысл |
|----------|--------|
| `paint(v)` | Пометить вершину `v` красной (повторная краска той же вершины игнорируется там, где это предусмотрено). |
| `query(v)` | Вернуть расстояние (число рёбер) от `v` до **ближайшей** красной вершины. Если красных нет — специальное значение (в бенчмарке не встречается: первый шаг всегда `paint`). |

Генерируется последовательность из `q` операций. Первый шаг **всегда** `paint` (случайная вершина). Дальше с вероятностью `p` — `paint`, иначе — `query` (вершины тоже случайные, фиксированный `seed`).

---

## Сравниваемые алгоритмы

На каждом тест-кейсе все три реализации прогоняют **одну и ту же** последовательность запросов. Ответы сверяются через **checksum**; при расхождении процесс завершается с ошибкой.

### 1. Centroid Decomposition (центроидная декомпозиция)

- **Построение:** один раз декомпозиция дерева, для каждой вершины — цепочка центроидов с расстояниями.
- **`paint(v)`:** обновить `best[c]` для всех центроидов `c` на пути `v` — **O(log n)**.
- **`query(v)`:** минимум `best[c] + dist` по цепочке — **O(log n)**.
- **Память:** структура цепочек + массив `best`.

На графиках: **сплошная линия**, «Centroid».

### 2. Naive BFS (наивный BFS)

Простейший эталон «как в лоб»:

- **`paint(v)`:** `red[v] = true`.
- **`query(v)`:** обнулить массив `dist` длины `n`, BFS из `v`, остановиться на первой красной.

Сложность запроса **минимум O(n)** из-за `fill(dist)`, даже если красная вершина рядом. Зато простая логика и малые накладные расходы; при большом числе красок BFS часто останавливается сразу.

На графиках: **пунктир**, «Naive BFS».

### 3. sqrt(n)-батчи + LCA

**Не** «чистый LCA» из слайда (им одним nearest red не решить). Это **гибрид**:

- Предобработка: дерево, бинарные подъёмы, LCA, `batch ≈ √n`.
- При накоплении `batch` новых красок — **полный multi-source BFS** по всем красным → массив `base_dist[]`, очистка `pending` — **O(n)**.
- **`query(v)`:** `min(base_dist[v], min по u ∈ pending dist_LCA(v,u))`. Между перестройками на каждый query перебирается `pending` (до ~√n вершин).

Амортизированно разумно на бумаге, но на практике при `q = 50 000` и частых `paint` много тяжёлых BFS по всему дереву — **может быть медленнее Naive BFS** (см. раздел «Как читать результаты»).

На графиках: **штрих-пунктир**, «sqrt(n)-батчи + LCA».

## Режимы бенчмарка

Задаются в `centroid_benchmark_advanced_verbose.cpp`, режим — первый аргумент EXE.

| Режим | Деревья | Нагрузки | Сетка `n` | `q` | Кейсов / seed |
|-------|---------|----------|-----------|-----|----------------|
| `smoke` | random, path | balanced | 300, 500 | 600 | 4 |
| **`quick` / `demo`** | **random, path** | read_heavy, balanced, update_heavy | **300 … 30 000, шаг 100** (~298 точек) | **50 000** | **~1 788** |
| `full` | random, path | те же | та же | 50 000 | ~1 788 |

### Типы деревьев

| `tree_type` | Описание |
|-------------|----------|
| `random` | Случайное дерево (Prüfer-подобная генерация). |
| `path` | Путь `0—1—2—…—(n-1)`. |

### Нагрузки (workload)

| `workload` | Вероятность `paint` | Смысл |
|------------|---------------------|--------|
| `read_heavy` | 0.01 | Мало красок, много запросов. |
| `balanced` | 0.15 | Сбалансированный режим; **на PNG по умолчанию**. |
| `update_heavy` | 0.50 | Много красок. |


## Требования

- **C++17**, компилятор `g++` (MinGW/MSYS на Windows).
- **Python 3** с пакетами: `pandas`, `matplotlib`, `numpy`.
- Windows: для замера памяти линкуется `-lpsapi` (Private Usage процесса).
